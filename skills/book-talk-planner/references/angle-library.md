# 讲书角度库（可扩展）

> 这是 `book-talk-planner` skill 的**核心资产**：从多个角度看一本「有故事的书」的视角维度库。
> 围绕「**人物 × 时代 × 利益 × 动机**」组织。每条视角固定 slot，供 skill 一致使用、供 `scripts/lint_angles.py` 机器校验。
> **契约与质量解耦**：结构契约（slot 齐全、id 唯一、无 banned 词）由 `lint_angles.py` 保证；视角本身的"够不够专业"靠自审。

---

## 一、视角登记（10 条内置）

每条格式固定，**标题逐字保留**：`### <id> · <中文名>`，下设 `- 看什么 / - 何时强 / - 钩子倾向 / - 示例` 四个 bullet。

### character-fate · 人物命运
- 看什么：关键人物的抉择与命运转折，性格/境遇如何一步步把他推到那个结局
- 何时强：书里有鲜明主角、重大抉择、命运反转时
- 钩子倾向：中（靠人物代入拉共鸣）
- 示例："他手里本来攥着一副赢牌，却自己一张张拆掉了"

### era-context · 时代语境
- 看什么：把人和事放回当时的历史/社会背景——为什么"只能那样"，而非简单的善恶
- 何时强：时代感强、人物被时代裹挟时
- 钩子倾向：中
- 示例："不是他蠢，是那个年代所有人都只能这么选"

### interest-game · 利益博弈
- 看什么：表面是道德/情感/恩怨，底层是经济利益与利益结构的博弈
- 何时强：有利益冲突、钱权纠葛、资源争夺时
- 钩子倾向：高（"原来都是为了钱"的揭秘感）
- 示例："这哪是兄弟反目，这是一条商路归谁的问题"

### power-game · 权力博弈
- 看什么：谁与谁博弈、力量对比如何变化、制度如何塑造每个人的行为空间
- 何时强：涉及权力、制度、组织内斗时
- 钩子倾向：高
- 示例："皇帝也不过是这台机器里一个换不了的零件"

### psych-motive · 心理动机
- 看什么：人物为什么真要这么做——人性的幽暗与光辉、恐惧与欲望
- 何时强：人物心理复杂、行为有隐秘动机时
- 钩子倾向：中
- 示例："他不是不怕死，他是更怕活着没人记得他"

### counter-intuitive · 反常识
- 看什么：大众以为是 A，书里揭示其实是 B——颠覆固有认知
- 何时强：书里有反直觉的结论、被误解的人/事时
- 钩子倾向：**最高**（前 3 秒最强的钩子来源）
- 示例："你以为他是个昏君，其实他是被史书写黑的最会算账的皇帝"

### then-now · 古今对照
- 看什么：历史照进现实——和今天的职场/社会/人情一一对应，让普通人有共鸣
- 何时强：能与当下经验类比的情节时
- 钩子倾向：高（共鸣驱动转发）
- 示例："明朝那场站队，和你公司里那次选边，是一模一样的牌"

### cause-chain · 因果链·蝴蝶效应
- 看什么：一个小决定/小事件如何一步步引发远超预期的后果
- 何时强：有清晰的因果链条、蝴蝶效应时
- 钩子倾向：高
- 示例："他随口答应的那一顿饭，五年后要了三千人的命"

### ensemble · 众生群像
- 看什么：不只看主角——配角、边缘人、被忽略的小角色如何折射整个时代
- 何时强：群像丰富、小人物命运有代表性时
- 钩子倾向：中
- 示例："书里只给了她三行字，但她才是那个时代最真实的脸"

### regularity · 规律模型
- 看什么：这件事背后有没有可提炼、可复用的规律或模型，让人看完能用
- 何时强：情节里有可抽象的底层规律时
- 钩子倾向：中（偏知识型受众）
- 示例："看懂这条，你就看懂了过去两千年所有改朝换代"

## 二、如何使用这个库

- **不是照念**：每条视角是**假设性工具**。分析一本书时，逐条判"适用 / 调整 / 不用"，只对"适用"的展开；不要硬凑 10 个角度都用上。
- **覆盖而非堆砌**：一本书选 **3-8 个值得讲的选题点**即可，但角度要**分散**（覆盖 ≥ 3 个不同视角），避免全挤在"人物命运"一个维度。
- **钩子优先级**：选题若能命中 `counter-intuitive` / `then-now` / `interest-game`（钩子倾向高/最高），爆款潜力更大——优先纳入。
- **连回具体内容**：每个选题点要落到书里**具体的人和事**，不能空讲"道理"。

## 三、加一个视角（最小步骤，零逻辑改动）

1. 在「视角登记」里按固定格式新增一条：`### <kebab-id> · <中文名>` + 四个 bullet（看什么 / 何时强 / 钩子倾向 / 示例）。
   - `<kebab-id>` 用小写字母+连字符，须与已有 id 不重复。
   - 「钩子倾向」填 `低 / 中 / 高 / 最高`。
2. 跑 `python3 scripts/lint_angles.py`，结构、id 唯一、banned 词全过才算加成功。
3.（无需改 SKILL.md 或任何脚本逻辑）新增的视角会自动被 `validate_outline.py` 的 `O-ANGLE` 规则接受。

## 四、Banned 词（写视角与选题时禁用，lint 会扫）

`体验好` `功能完善` `功能强大` `适当处理` `待定` `待补` `后续再说` `良好体验` `非常重要` `很关键` `等等` `TBD` `TODO`

→ 要表达"很强/很关键"时，改成**具体可核**的说法（量化、具名、因果），见上面各条示例的写法。
