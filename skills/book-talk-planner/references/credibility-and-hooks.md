# 可信度标注 + 钩子/标题/封面方向（模块 G + H）

> Checklist 第 6、7 步用。
> **G 可信度**：讲书涉及历史/经济/数据，LLM 易编——对每个论断标可信度，降"讲错翻车"风险（门 `O-CRED`）。
> **H 钩子/标题/封面**：抖音前 3 秒定生死，每个选题点给钩子方向 + 标题方向 + 封面方向。

---

## 一、哪些论断必须标可信度

只要 topic 的 `key_points` / `hook_directions` 里出现以下任一，就**必须**进 `credibility[]` 逐条标注：

- **具体历史年代/事件**（"万历二十九年…""1929 年大萧条…"）
- **人物言行/关系**（"他做了 X""两人其实是 Y 关系"）
- **经济/数据**（"占 GDP 的 X%""死了 N 万人"）
- **因果判断**（"正是因为 A 才导致 B"）—— 因果是幻觉重灾区，尤其要标。

纯观点/感受/金句方向（"这让人想到…"）不需要标。

## 二、level 怎么定

| level | 判据 |
|-------|------|
| `credible` | 全书原文有据、或史实公认常识（如"万历长期不上朝"） |
| `uncertain` | 有说法但非通行共识 / 是作者一家之言的史观（如黄仁宇"大历史观"的特定解读） |
| `unverified` | 凭 LLM 记忆、无原文支撑、具体数字/年代拿不准 |

`reason` 写清判据来源（"全书核心议题，史实公认" / "作者一家之言，讲时需注明" / "凭记忆，成稿前需核实出处"）。

## 三、输入风险联动（门 `O-CRED` 硬卡）

- `input_mode == title_only`（凭 LLM 记忆）→ `unverified`/`uncertain` 占比应**显著高于**有摘要/全书输入。具体年代/数据一律倾向 `unverified`。
- `full_text` → 能在原文找到出处的标 `credible`，找不到的才降级。
- 这条是"不想出错"的核心保障：宁可多标 unverified，也不让可疑事实混进成稿当定论。

---

## 四、钩子方向（hook_directions，每点 1-2 条）

**只指方向，不写成品句**（成品逐字稿归下游）。常用钩子套路（抖音通行，2025 仍有效）：

- **反常识**："你以为 A，其实是 B"（钩子最强，优先）
- **悬念**："他做了一个决定，五年后…"
- **数字**："三百人里只有他活下来"
- **冲突**："亲兄弟，为了 X 反目"
- **金句先行**：把最扎心的结论甩开头

每点给 **1-2 个**钩子方向（可备选），写明用哪种套路 + 钩什么。

## 五、标题方向（title_directions，每点 2-3 条）

钩子型标题，**2-3 个**备选。要求：
- 口语、有信息差、不书名化（避免"《X》读后感"）；
- 能独立成立（不点开也知道大概讲什么）。

## 六、封面方向（cover_direction，每点 1 条）

一句话描述封面画面 + 主视觉字。要求：
- 画面有冲突感/悬念（剪影、对比、锁链、天平等意象）；
- 主视觉字 ≤ 8 字，是钩子不是书名。

---

## 自检（交付前过）

- 每个有史实/经济/数据论断的 topic，`credibility` 是否逐条覆盖？
- `title_only` 输入下，unverified/uncertain 占比是否够高？
- 每点钩子方向是否点了**套路**（不是干写一句话）？
- 标题是否口语化、有信息差（不像论文标题）？
