# 写作法则：结构 / 逻辑 / 表达

> 配合 doc-blueprint skill 的 Checklist 第 5 步（逐节写作）。
> 写作法则的**主战场在 doc-blueprint**（写作层）。分三类：A 结构/领域法则、B 逻辑/论证法则、C 表达/风格法则。
> 它们与 `expressiveness.md`（表现力=D 类，偏"用什么呈现"）互补：本文件管"内容怎么写得专业"，那里管"内容用什么形式呈现"。
> 写作时**应用**，校验时（`validation-rules.md` R-W 系列）**强制**。

---

## A. 结构 / 领域法则（从文档类型库派生，校验 R-S）

正文结构不凭空发明，从 `doc-type-library/<doc_type>.md` 的必备章节派生。

**法则**：
1. **必备章节齐全** —— 每类文档有其"缺了就不专业"的章节（见各模板）。例：复盘无根因=不合格；RFC 无备选方案=不合格；PRD 无可度量成功指标=不合格。
2. **每节只承一个主旨** —— 一节讲清一件事；多主旨拆节。
3. **结论先行（金字塔）** —— 每节首句/首段给该节结论，再展开支撑。整篇也是：摘要/结论在前，细节在后。
4. **章节有 intent** —— 每节能说清"为什么存在"（写进 `<!-- doc:section intent= -->`）。

**反模式**：
- 缺必备章节（复盘没写根因，只罗列现象）。
- 一节塞多个主旨，读者抓不住重点。
- 细节先行，结论藏在末尾（决策者读到一半就放弃）。

## B. 逻辑 / 论证法则（校验 R-W）

**法则**：
1. **论断必有证据** —— 每个论断（"效果很好""性能足够"）要么引用 `datasets`/`references`，要么显式标"假设"（`confidence: assumed`）。无证据的最高级（"最优""最好""业界领先"）禁止。
2. **决策必有理由 + 备选** —— 每个 `[!decision]` 必带理由；技术/方案决策至少列一个备选方案与权衡（为什么没选它）。
3. **因果链显式** —— 从现象到根因、从根因到改进，链条不跳步。用"因为→所以""导致→因此"显式连接。
4. **数字带口径** —— 每个数字写明口径与时点（`datasets[].caliber`）；不带口径的数字不可信。
5. **范围与边界清楚** —— 明确"适用什么、不适用什么"，不泛化。

**反模式**：
| ❌ | ✅ |
|----|----|
| "我们的方案性能优秀" | "P99 延迟从 {{d:p99_before}} 降到 {{d:p99_after}}（压测，见 references.r2）" |
| 决策："决定用 Kafka" | 决策 + 理由（吞吐需求 X）+ 备选（RabbitMQ，不选因吞吐不足） |
| "导致了故障" | "连接池打满 → 请求排队 → 超时级联 → 用户支付失败" |
| "营收很高" | "Q3 营收 {{d:rev_q3}} 万元（GAAP，含 X 不含 Y）" |

## C. 表达 / 风格法则（自审）

**法则**：
1. **结论先行，再展开** —— 段落首句给要点；金字塔从上到下。
2. **具体而非抽象** —— 用"3 天内回复率提升 40%"，不用"大幅提升"。
3. **主动语态、删冗词** —— "我们决定"，不是"决定被做出了"；删"基本上/其实/可以说"。
4. **术语对受众** —— 对内用行话；对外/跨职能要解释。受众来自 brief。
5. **MECE + 平行结构** —— 并列项互斥穷尽、同层级同句式（`- 名词：解释` 全用同结构）。
6. **一句话一意** —— 长句拆短；一段 ≤ 5 句。

**反模式**：
| ❌ | ✅ |
|----|----|
| "为了更好地服务于广大用户的需求，我们进行了优化" | "我们把下单步骤从 5 步减到 3 步" |
| 并列项一句长一句短 | 统一为 `- 标题：说明` |
| 满篇"基本上/其实/可以说" | 删 |
| 对外文档堆术语 | 首次出现解释 |

## D. 投影 / 格式法则（属 doc-render，此处仅声明边界）

图表/表格/状态/代码**在各后端怎么渲染**（mermaid vs Confluence 宏、MD 表 vs Confluence 表）是 `doc-render` 的事，**不在本 skill**。本 skill 只保证：每个块标对了 block kind、造对了数据、数字同源——渲染器据此投影。

> 边界：当思路滑向"Confluence 里这个 callout 用 info 宏还是 warning 宏"时，拉回——本 skill 只定"这是 warning 类警示"，具体宏由 `doc-render`。
