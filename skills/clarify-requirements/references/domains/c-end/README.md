# C 端通用 领域知识

> 树路径：c-end
> 继承：无（根节点）
> 子类：`education`（教育）/ `tools`（工具类）/ `games`（游戏类）。命中子类请下钻；未命中子类、或子类暂未内置知识时，**回退到本节点**。

## 匹配信号
- 给**普通消费者**用，个人用户；非企业内部、非开发者工具。
- 形态多为 App / 网站 / 小程序。

## 核心价值要素
解决普通消费者的某个日常需求或欲望。通用的"不可缺"：① **低摩擦**——快速达成核心动作（注册、上手、完成主任务都要快）② 一句话能讲清的核心价值主张。

## 业界标配功能（通用层，各子类会再叠加）
注册 / 登录（或游客试用）、个人中心、消息或通知、设置、帮助 / 反馈、新用户引导。

## 分析侧重（C 端通用镜头）
- **用户旅程**：从首次接触到达成目标的每一步。
- **留存与激励**：是什么让人明天还来？（打卡 / 连续 / 提醒 / 内容更新）。
- **低摩擦录入与核心路径**：完成主任务要几步？
- **心流体验**：难度与能力匹配，不打断。

## 业界标杆做法
看该品类的**头部竞品**（应用商店榜单 / 头部产品）在关键环节怎么做。

## 常见陷阱与反模式
- 注册 / 登录摩擦大 → 首日流失。
- 核心路径埋得太深 → 用户找不到主功能。
- 新用户无引导 → 不知道能干嘛，流失。
- 激励缺失或过度 → 留不住 或 打扰。

## 本节点专属澄清
- 主要给谁用、什么场景下打开它？
- 完成核心任务现在要几步？能否更短？
- 靠什么让用户明天还来？
