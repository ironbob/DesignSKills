# 工具类 领域知识

> 树路径：c-end > tools
> 继承：`c-end`（低摩擦 / 核心路径 / 留存仍适用）
> 使用前先读父类：[`README.md`](README.md)

## 匹配信号
- 关键词：扫描 / PDF / 计算器 / 天气 / 时钟 / 番茄钟 / 清理 / 格式转换 / 压缩 / OCR / 密码管理 / 翻译（工具）/ 文件管理 / 记事。
- 场景：临时要完成一个明确小任务——查天气、扫一份文件、转个格式、记一笔、专注 25 分钟。
- 判据：核心是"完成一个动作后用完即走"，不是消费内容 / 社交 / 学习。

## 继承要点（速览）
继承 c-end 通用（低摩擦 / 核心路径）；工具类的增量是——**"用完即走"的单一任务闭环**，以及**变现与体验的艰难平衡**。

## 核心价值要素
让用户**快速完成一个单一、明确的小任务**（查一下、扫一下、转一下）。不可缺：① 低摩擦（打开即用、步骤极少）② 任务一次成功（不用返工）。工具是"用完即走"的极致，和内容 / 社交类相反。

## 业界标配功能
- **免登录即用** —— 游客 / 无需注册直接用核心功能（工具类注册墙是死亡信号：替代品一搜一堆）。
- **零摩擦入口** —— 桌面小组件 / 快捷指令 / 分享菜单接入 / 通知中心（工具要"召之即来"）。
- **离线可用** —— 计算器、扫描、PDF 等关键场景常无网（离线缺失 → 场景用不了）。
- **批量处理** —— 扫多页、转多个文件、清多项（高频用户的效率分水岭）。
- **历史 / 最近** —— 最近扫的、最近转的（省去重复找文件）。
- **导出 / 分享结果** —— 到相册 / 邮件 / 其他 App（结果流不出去就没用）。
- **核心功能精准** —— 扫描要边框矫正 + OCR；PDF 要合并拆分；翻译要准确（一次失败，永久弃用）。

## 业界标杆做法
- **扫描全能王（CamScanner）**：边缘矫正 + OCR + 云同步，把"扫一下"做到专业级。
- **1Password**：生物解锁 + 跨端同步 + 自动填充，把高频小动作做到无感。
- **快捷指令（Shortcuts）/ IFTTT**：把单一工具编进系统流程，可组合、可自动化。
- **计算器 / 时钟**：零启动成本，"用完即走"的极致标杆。

## 常见陷阱与反模式
- **注册墙 / 强制登录** → 用户秒退（工具类头号死因，替代品太多）。
- **核心功能不精准**（OCR 错、转换丢格式、清理误删）→ 一次失败，永久弃用。
- **广告打断核心动作** → 工具是"用完即走"，广告直接杀体验。
- **单一工具做臃肿**（什么都塞）→ 失去"快"的核心，被更专的工具替代。
- **免费版阉割核心功能** → 流失或差评（变现与体验冲突）。
- **无离线 / 无批量** → 关键场景或高频用户用不顺。

## 本节点专属澄清
- 核心任务是**哪一个单一动作**？（一句话讲清）
- 要不要**登录**？（强烈建议免登录）
- 有没有**离线**场景？
- 高频用户要不要**批量**？
- **变现方式**（广告 / 订阅 / 买断）？怎么不伤核心体验？
- 结果怎么**导出 / 分享**？
