## 匹配信号
- 关键词：学习 / 课程 / 练习 / 考试 / 背单词 / 教学 / 知识点 / 掌握 / 刷题 / 复习。
- 场景：用户为"学会 / 记住 / 掌握"某项能力而反复使用。

## 继承要点（速览）
C 端通用仍适用；但教育比普通 C 端多一个硬内核——**教学法与效果**。

## 核心价值要素
让用户"**学会 / 记住 / 掌握**"某项能力，而不只是"用过"。不可缺：① **学练测评闭环**（学 → 练 → 测 → 评 → 复盘，断一环就失效）② **掌握度反馈**（让用户知道自己"会不会"，而非"练没练"）。

## 业界标配功能
- 学练测评闭环各环节对应的功能（视频 / 图文讲解 → 题目练习 → 测验 / 考试 → 反馈评分 → 错题 / 复盘）。
- **间隔重复 / 复习调度（SRS）**——决定记忆类产品的成败。
- 进度追踪与**掌握度**（知识点级、薄弱点定位）。
- **激励与留存**：打卡 / 连续天数 / 徽章 / 排行榜 / 心流难度调节（学习反人性，激励是生存问题，非锦上添花）。
- 内容结构：课程 / 章节 / 知识点 / 难度分级。
- 错题 / 薄弱点收录与重练。
- 多角色：学习者（K12 还常有**家长 / 老师**）。

## 业界标杆做法
- **Duolingo**：游戏化句型 + 即时反馈 + streak 连续（留存教科书）。
- **Anki**：间隔重复（记忆类地基）。
- **Khan Academy**：掌握学习（mastery learning），不达标不解锁下一关。
- 降低启动阻力：首页"继续学习"大卡片。

## 常见陷阱
- 学习页塞社交 / 广告 → 打断心流。
- 结果页无"下一步" → 死胡同。
- 一次性铺题海 → 不如单题聚焦 + 即时反馈。
- **闭环断裂**：只练不测 / 只测不复盘 → 学不会。
- **无掌握度反馈** → 练了不知道会不会。
- 激励过度 → 变打扰，稀释学习密度。

## 本节点专属澄清
- 学什么内容？（据此下钻 k12 / language / career）
- 学完怎么算"**掌握**"？（效果衡量指标——教育头号难题）
- 学练测评闭环全不全？哪一环可能断？
- 留存靠什么机制？
- 有没有家长 / 老师角色？
